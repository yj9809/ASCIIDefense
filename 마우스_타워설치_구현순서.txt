마우스 기반 타워 설치 구현 아이디어 (순서)

1. 입력-게임로직 분리
- Input은 "클릭됨" 이벤트만 전달하고, 설치 판단은 GameLevel/BuildSystem에서 처리
- 테스트/디버깅을 쉽게 하기 위한 구조

2. 설치 파이프라인 고정
- 클릭 -> 타일 변환 -> 유효성 검사 -> 설치 실행
- 단계를 함수로 분리해서 버그 위치 추적이 쉽게 구성

3. 유효성 검사는 사유 코드로 반환
- 예: Ok, OutOfBounds, OnPath, Occupied, NotEnoughGold, BlocksAllPaths
- UI 피드백과 로그에서 동일 코드 재사용

4. 경로 차단 체크는 임시 배치로 검증
- 후보 타일을 임시로 막은 뒤 A*/BFS로 goal 도달 가능 여부 확인
- 불가능하면 롤백

5. 고스트(미리보기) 먼저 만들기
- 클릭 전 설치 가능/불가 상태를 색상으로 표시
- 사용자 경험 개선 + 디버깅 편의성 향상

6. 빌드 모드 상태머신 도입
- 예: Normal, Build, Sell, Upgrade
- 현재는 Build만 먼저 적용해도 확장성 확보

7. 데이터 기반 타워 정의
- 비용, 사거리, 공격속도, 문자/색상 등은 테이블(구조체/파일)로 분리
- 하드코딩을 줄이고 밸런싱 작업 속도 향상

8. 최소 로그 포인트 추가
- 클릭 좌표, 변환 타일, 실패 사유, 설치 성공 이벤트를 한 줄 로그로 기록
- "설치가 왜 안 되는지"를 빠르게 파악 가능
